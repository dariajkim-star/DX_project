## 1. 대상 (Who)

| 군 | 채널 | 목표 |
|---|---|---|

목표 표본: **최소 30명** (`04_segmentation` 가동 하한), 이상 50~100명.

## 2. 문항 (What) — 5블록 · 15문항 내외 · 5분 컷

> ⚠️ 순서 고정: **③ Pain 회상 → ⑤ 지불 의사** 순서를 지킬 것 (Pain을 떠올린 뒤 물어야
> 지불 의향이 실측됨 — Grumbal 원칙). ⑤를 앞에 두면 무근거 낙관 수치가 나온다.

### ① 스크리닝 (2)
1. 보유 중인 LG 스마트가전(ThinQ 연동 가능) 수는? `0 / 1 / 2~3 / 4+` — 0은 설문 종료
2. ThinQ 앱 사용 빈도는? `설치만 / 월 1회 이하 / 주 1~2회 / 거의 매일`

### ② 인구통계 (5)
3. 연령대 `20대 이하 / 30대 / 40대 / 50대 / 60대 이상`
4. 성별 `여 / 남 / 응답 안 함`
5. 가구 형태 `1인 / 부부 / 자녀 있는 가족 / 부모님과 거주 / 기타`
6. 만 7세 이하 자녀 `있음 / 없음`
7. 주거 형태 `아파트 / 오피스텔·빌라 / 단독 / 기타`

### ③ Pain 검증 (3 — 각 5점 척도: 전혀 없었다 ~ 매우 자주)
8. **[P-1]** 앱·서버 문제로 가전을 제어하지 못한 경험이 있다
9. **[P-2]** 앱 업데이트·재설치·공유기 교체 후 기기를 다시 등록하거나 설정을 다시 한 경험이 있다
10. **[P-3]** 회원가입·약관 동의·개인정보 요구가 부담스러웠던 적이 있다

### ④ 워치·수용도 (3)
11. 스마트워치 보유 `없음 / 애플 / 갤럭시 / 가민 / 기타`
12. 주로 밤(22시 이후)에 가전을 앱으로 제어한다 `5점 척도`
13. "앱을 열지 않아도, 손목의 워치가 내 설정을 갖고 있어 집이 알아서 반응한다면 사용하겠다" `5점 척도`

### ⑤ 지불 의사 (2)
14. 위 기능이 제공된다면 월 구독료 지불 의향은? `없음 / 3천원 / 5천원 / 1만원`
15. (선택) ThinQ 앱에서 가장 불편했던 경험 한 가지 — 주관식

## 3. FEATURE_COLUMNS 매핑 (04_segmentation 연동)

```python
FEATURE_COLUMNS = [
    "연령대",        # 문3: 1~5 서열 인코딩
    "가구형태",      # 문5: 원핫 또는 자녀유무로 축약
    "자녀유무",      # 문6: 0/1
    "LG가전수",      # 문1: 0/1/2/3 서열
    "앱사용빈도",    # 문2: 1~4 서열
    "P1경험",        # 문8: 1~5
    "P2경험",        # 문9: 1~5
    "P3부담",        # 문10: 1~5
    "워치보유",      # 문11: 0/1 (기종은 별도 컬럼)
    "야간사용",      # 문12: 1~5
    "온바디수용도",  # 문13: 1~5
    "지불의사",      # 문14: 0/3/5/10 (천원 단위)
]
```
K-Means 검증 포인트: **P1 세그먼트(유자녀·야간사용↑·P1경험↑)가 실제로 분리되는가** +
그 세그먼트의 온바디수용도·지불의사가 타 세그먼트보다 높은가.
