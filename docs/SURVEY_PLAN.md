## 0. 검증 가설
- **H1 (P1 실재)**: 유자녀·야간사용↑·P1경험↑ 세그먼트가 K-Means에서 분리된다
- **H2 (점유형태, 2026-07-21 다리아 제안)**: **전세·월세일수록, 이사 계획이 가까울수록 온바디
  프로필 수용도가 높다** — 전세 거주자는 "이사가 예정된 사람"이라 프로필 이식성 니즈가 구조적.
  참이면 1차 GTM 타겟 = 전세 신혼부부. (12월 신혼 라이프사이클 Agent·2월 이사 초기화 문제의식과 연결)
- 지역은 가설 없음 → **수집·군집 모두 제외** (지역≠서버 장애 요인, 표본 30~50에 무신호 변수 금지
  — 04 v2.2 화이트리스트 원칙)

필수 스크리닝: **LG 스마트가전 1대 이상 + ThinQ 설치 경험** (문1에서 0대면 종료)

## 3. FEATURE_COLUMNS (확정 후보 — 인코딩 포함)

```python
FEATURE_COLUMNS = [
    "LG가전수",      # 문1: 1/2/3 서열
    "앱사용빈도",    # 문2: 1~4
    "연령대",        # 문3: 1~5
    "자녀유무",      # 문4에서 파생: 0/1 (미취학 여부는 별도 파생 가능)
    "점유형태_전월세", # 문5에서 파생: 자가0/전월세1  ← H2
    "이사계획",      # 문6: 있다1/없다0/모름0.5     ← H2
    "구매계기_혼수",  # 문7에서 파생: 0/1
    "P1경험", "P2경험", "P3부담",   # 문8~10: 1~5
    "워치보유",      # 문11a: 0/1
    "야간사용",      # 문11b: 1~5
    "온바디수용도",  # 문12: 1~5
    "지불의사",      # 문13: 0/3/5/10
]
```
검증 포인트: H1 세그먼트 분리 + H2 상관(점유형태·이사계획 × 온바디수용도·지불의사).

## 4. 설문 이후 — ThinQ Village (합성 소비자 패널, 2026-07-21 확정)

목적: **MoA 역량 어필** — 강의의 MoA를 "실측 기반 합성 소비자 패널"로 응용·확장했다는 서사.
순서가 핵심: **실설문이 먼저, 에이전트는 그 결과로 조련** (arXiv 2411.10109 방법론 준용 —
실측 없이 만든 에이전트는 GPT 사전확률일 뿐이라는 파티 합의).

1. 실설문 30~50명 → H1·H2는 **실측으로만** 검증 (불변 원칙)
2. `04_segmentation` K-Means 세그먼트 프로파일(z-score+대표 응답)을 **세그먼트별 에이전트의
   시스템 프롬프트로 주입** (+실리뷰 인용 — B 에이전트 주입과 동일 패턴)
3. 에이전트 용도: 설문으로 못 묻는 what-if(가격 변동·기능 조합), 시나리오 반응(이사 당일 서사),
   에이전트 간 상호작용 시뮬레이션
4. **홀드아웃 재현 테스트**: 문항 2개를 학습에서 제외 → 에이전트 응답 vs 실응답 분포 비교로
   자체 재현율 산출 (85% 논문 인용 시 "우리 규모에선 n%"로 정직 표기)
5. 산출물 라벨: `synthetic_panel` 워터마크, `survey.csv`와 파일 절대 분리 (계보 원칙)
6. MoA 편입: 시뮬레이션 결과는 G(Claude)가 교차 검증 → 사람 검수 체크리스트
