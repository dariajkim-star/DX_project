## 1. 프로젝트 주제 (v4-final)

> **"집이 나를 따라온다 (Home Follows You)"**
> — ThinQ VOC가 실증한 클라우드 종속 Pain을, 워치에 담긴 Portable Home Profile로 풀다

- **발견**: ThinQ 리뷰 5,000건 DX 파이프라인 분석 + MoA 교차 검증(예정) — 클라우드 SPOF(부정 리뷰 42.5%)·설정 휘발/재등록(0.9%)·개인정보 불신 실증
- **진단**: 공통 원인 = 홈 프로필의 클라우드·앱 종속
- **처방**: 캐리어 중립(애플·가민·샤오미 — 삼성 갤럭시워치 폐쇄 진영에 대한 개방 생태계 전략) **온바디 홈 프로필** + BLE/UWB 로컬 제어. 보안 위협 모델(릴레이 공격·분실 폐기) 1장 포함
- **실증 데모**: 가민 실기기(보유) → BLE → 가전 시뮬레이터 — "서버 없이 야간 모드 전환"
- **기능 레이어**: 에너지 통합 모니터링 (토픽6 직접 요구, 26건)
- **v2 비전 레이어**: Life Event Agent(집 상태 전환) / 웰니스 확장 슬롯(수면·활동 → 체성분은 스키마 예약 필드만. 진단 기능 배제 — 의료 규제)
- **스트레치**: SmartThings 500건 대조, 설문 기반 세그먼트 검증

### 아이디어 계보 (뇌피셜 방어)
2025.12 후보 8개(Life Event Agent·Portable Profile 등) → 2026.07 실VOC 분석이 ⑧ Portable Profile을 근거로 채택(단독 정량은 0.9%로 소수 — "주요 Pain"이 아니라 "실증된 기회 영역"), ③ Life Event는 비전으로 강등, ⑤⑦②④⑥은 VOC 미지지로 제외. **"VOC가 문제를 줬고, 해법은 우리의 설계 의사결정"** — 해법 선택은 트레이드오프 표(앱 개선 vs 로컬 허브 vs 온바디)로 제시한다.

## 2. CX Pain Point (Mary 3단 확정 프레임)

**"Pain Point는 ThinQ 앱 VOC에서 뽑았다"** — 발견까지가 데이터, 진단부터는 분석, 처방부터는 설계.

### 2.1 발견 (실증 — 전량 painpoints.csv 출처, 언급률은 전체 부정 리뷰 기준)

| Pain | 정량 | 대표 인용 |
|---|---|---|
| **P-1 클라우드 SPOF** — 앱/서버 죽으면 가전 제어 불가 | 토픽0 42.5% + 야간 먹통 토픽 계열, 2026.07 사태 시 월 리뷰 10~30배 폭주·1점 73% | "앱이 갑자기 실행이 안되서 삭제하고 다시 설치했는데 열리지가 않네요" |
| **P-2 설정 휘발·재등록** — 홈 설정이 사용자 자산으로 보존 안 됨 | 37건(0.9%) — 소수이나 경쟁 열위 실증 포함 | "핸드폰 초기화하고 재설치하니 제품이 전부 없어짐. **삼성 껀 그대로인데**" / "등록해둔 제품들이 자주 없어지고 지워져서 다시 등록해야 함" |
| **P-3 진입장벽·개인정보 불신** | 약관 루프 4.8% + 로그인 3.5% + 개인정보 0.4% ≈ 9% | "내 집에 있는 제품들 연결해서 쓰는 건데 굳이 회원가입을 강요하는 이유가 뭡니까" |

한계 명시: 표본은 최신순 5,000건으로 2026.07 사태 기간(1,556건, 1점 73%)이 과대 표집됨 — 시기 분리 분석을 본문에 병기. 사태 이전 3.4년 구간 1점 비율은 30%.

### 2.2 진단 (분석)
세 Pain의 공통 원인: **홈 프로필(기기 등록·설정·루틴)이 클라우드 계정과 앱에 종속된 구조.** 서버가 죽으면 집이 죽고(P-1), 앱이 리셋되면 설정이 죽고(P-2), 프로필이 서버에 있으니 계정·약관·개인정보 수집이 강제된다(P-3).

### 2.3 처방 (설계 의사결정 — 데이터 아님을 명시)
홈 프로필을 사용자 온바디 디바이스(워치, 캐리어 중립)에 귀속 + 가전과 BLE/UWB 로컬 통신. P-1(서버 무관 동작)·P-2(프로필이 사용자를 따라다님)·P-3(서버가 원본 미보유 = 프라이버시 셀링포인트)을 단일 아키텍처로 해소.
