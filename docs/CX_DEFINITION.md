## 1. 프로젝트 주제 (v4-final)

> **"집이 나를 따라온다 (Home Follows You)"**
> — ThinQ VOC가 실증한 클라우드 종속 Pain을, 워치에 담긴 Portable Home Profile로 풀다

- **발견**: ThinQ 리뷰 **17,446건** + 대조군 SmartThings 24,805건 + 네이버 32,183건 = **74,434건** 분석 (MoA 교차 검증 예정) — 클라우드 SPOF(1점 비율 30.0%, 대조군 13.5%)·설정 휘발/재등록(1.37%, 대조군 0.67%)·개인정보 불신 실증
- **진단**: 공통 원인 = 홈 프로필의 클라우드·앱 종속
- **처방**: 캐리어 중립(애플·가민·샤오미 — 삼성 갤럭시워치 폐쇄 진영에 대한 개방 생태계 전략) **온바디 홈 프로필** + BLE/UWB 로컬 제어. 보안 위협 모델(릴레이 공격·분실 폐기) 1장 포함
- **실증 데모**: 가민 실기기(보유) → BLE → 가전 시뮬레이터 — "서버 없이 야간 모드 전환"
- **기능 레이어**: 에너지 통합 모니터링·루틴 개선·스마트진단 정확도 (본수집 기능요청 토픽에서 실증)
- **v2 비전 레이어**: Life Event Agent(집 상태 전환) / 웰니스 확장 슬롯(수면·활동 → 체성분은 스키마 예약 필드만. 진단 기능 배제 — 의료 규제)
- **스트레치**: ~~SmartThings 500건 대조~~ → **완료(24,805건 대조 수행)**, 설문 기반 세그먼트 검증

### 아이디어 계보 (뇌피셜 방어)
2025.12 후보 8개(Life Event Agent·Portable Profile 등) → 2026.07 실VOC 분석이 ⑧ Portable Profile을 근거로 채택(본수집 기준 1.37% — 파일럿 0.9%보다 실체가 크고 **경쟁 대비 2.0배 열위**가 확인되어 "실증된 차별화 기회 영역"으로 승격), ③ Life Event는 비전으로 강등, ⑤⑦②④⑥은 VOC 미지지로 제외. **"VOC가 문제를 줬고, 해법은 우리의 설계 의사결정"** — 해법 선택은 트레이드오프 표(앱 개선 vs 로컬 허브 vs 온바디)로 제시한다.

## 2. CX Pain Point (Mary 3단 확정 프레임)

**"Pain Point는 ThinQ 앱 VOC에서 뽑았다"** — 발견까지가 데이터, 진단부터는 분석, 처방부터는 설계.

### 2.1 발견 (실증 — 전량 painpoints.csv 출처, 언급률은 전체 부정 리뷰 기준)

전량 재분석 완료(2026-07-21, run_id `20260721_161447`): ThinQ **17,446건 전량**(2015-11~2026-07),
딥러닝 감성 부정 **7,625건**, BERTopic 15토픽. 파일럿(†) 수치 전부 본수집 기준으로 갱신됨.

| Pain | 정량 (본수집 확정) | 대표 인용 |
|---|---|---|
| **P-1 클라우드 SPOF** — 앱/서버 죽으면 가전 제어 불가 | **토픽0(연결·에어컨·업데이트) 43.4%** — 파일럿 42.5%와 일치. **11년 전 구간에서도 유지 = 2026.07 사태 의존 아님 입증**. 1점 비율 30.0%(대조군 13.5%), 사태 시 월 리뷰 10~30배 폭주 | "앱이 갑자기 실행이 안되서 삭제하고 다시 설치했는데 열리지가 않네요" |
| **P-2 설정 휘발·재등록** — 홈 설정이 사용자 자산으로 보존 안 됨 | **1.37% (235건 / 17,117)** — 대조군 SmartThings 0.67%(166건)의 **2.0배** (정규식 측정 — 토픽11에도 "다날아갔네_등록안돰"으로 독립 출현) | "업데이트했더니 등록했던거삭제되고 다시 등록도안됨" / "강제업데이트하면 기존연결 다 풀리고" / "핸드폰 초기화하고 재설치하니 제품이 전부 없어짐. **삼성 껀 그대로인데**" |
| **P-3 진입장벽·개인정보 불신** | 약관·로그인 5.7% + 개인정보 0.5% ≈ **6.2%** (파일럿 9%에서 하향 — 사태 기간 약관 루프 과대표집이 희석됨) | "내 집에 있는 제품들 연결해서 쓰는 건데 굳이 회원가입을 강요하는 이유가 뭡니까" |

파일럿 대비 변동: P-1 42.5%→43.4%(안정), P-3 9%→6.2%(사태 편향 제거), P-2 1.37%(불변 — 정규식 측정이라 재분석 무관).
부차 발견: 기능 요청 토픽(루틴 개선 1.1%·속도 1.0%·스마트진단 부정확 0.4%)이 안정적으로 분리됨 — 제품 백로그 재료.

P-2 측정: 소실·재작업 명시 표현 정규식([compare_competitor.py](../dx_pipeline_v2.2/compare_competitor.py)),
눈검수 기반 정밀화 완료(잔여 오탐률 11.3%) — 경위는 [검증 미팅](meetings/2026-07-21_evidence-verification.md) §1.
배치 원칙: 헤드라인 1순위는 P-1, P-2는 "경쟁 열위가 명확한 차별화 근거".

**시간축 구조 증거**: 월별 1점 비율 오버레이(`out/compare_rating_trend.png`)에서 2017~2020 구간은
두 앱이 엉켜 있다가 **2021년부터 ThinQ만 지속 상승** — 일시적 장애가 아니라 구조적 열화이며,
"프로필의 클라우드 종속" 진단과 시간축이 일치한다.

한계 명시: ThinQ는 수집 상한 내 **전량**(2015-11~2026-07, 17,446건), SmartThings는 상한 절단
(24,805건, 2017-04~). 비교는 양쪽 공통 구간(2017-04~)으로 정합 후 수행. 2026.07 사태 구간이
포함되어 있으므로 시기 분리 병기 필요.

### 2.2 진단 (분석)
세 Pain의 공통 원인: **홈 프로필(기기 등록·설정·루틴)이 클라우드 계정과 앱에 종속된 구조.** 서버가 죽으면 집이 죽고(P-1), 앱이 리셋되면 설정이 죽고(P-2), 프로필이 서버에 있으니 계정·약관·개인정보 수집이 강제된다(P-3).

### 2.3 처방 (설계 의사결정 — 데이터 아님을 명시)
홈 프로필을 사용자 온바디 디바이스(워치, 캐리어 중립)에 귀속 + 가전과 BLE/UWB 로컬 통신. P-1(서버 무관 동작)·P-2(프로필이 사용자를 따라다님)·P-3(서버가 원본 미보유 = 프라이버시 셀링포인트)을 단일 아키텍처로 해소.
