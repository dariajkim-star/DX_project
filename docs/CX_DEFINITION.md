- **처방**: 캐리어 중립(애플·가민·샤오미 — 삼성 갤럭시워치 폐쇄 진영에 대한 개방 생태계 전략) **온바디 홈 프로필** + BLE/UWB 로컬 제어. 보안 위협 모델(릴레이 공격·분실 폐기) 1장 포함

- **기능 레이어**: 에너지 통합 모니터링 (토픽6 직접 요구, 26건)
- **v2 비전 레이어**: Life Event Agent(집 상태 전환) / 웰니스 확장 슬롯(수면·활동 → 체성분은 스키마 예약 필드만. 진단 기능 배제 — 의료 규제)

### 아이디어 계보 (뇌피셜 방어)

## 2. CX Pain Point (Mary 3단 확정 프레임)

**"Pain Point는 ThinQ 앱 VOC에서 뽑았다"** — 발견까지가 데이터, 진단부터는 분석, 처방부터는 설계.

### 2.1 발견 (실증 — 전량 painpoints.csv 출처, 언급률은 전체 부정 리뷰 기준)

| Pain | 정량 | 대표 인용 |
|---|---|---|
| **P-1 클라우드 SPOF** — 앱/서버 죽으면 가전 제어 불가 | 토픽0 42.5%†+ 야간 먹통 토픽 계열. **본수집 기준 ThinQ 1점 비율 30.0%** (SmartThings 13.5% — 2.2배), 2026.07 사태 시 월 리뷰 10~30배 폭주·1점 73% | "앱이 갑자기 실행이 안되서 삭제하고 다시 설치했는데 열리지가 않네요" |
| **P-2 설정 휘발·재등록** — 홈 설정이 사용자 자산으로 보존 안 됨 | **1.37% (235건 / 17,117)** — 대조군 SmartThings 0.67%(166건)의 **2.0배** | "업데이트했더니 등록했던거삭제되고 다시 등록도안됨" / "강제업데이트하면 기존연결 다 풀리고" / "핸드폰 초기화하고 재설치하니 제품이 전부 없어짐. **삼성 껀 그대로인데**" |
| **P-3 진입장벽·개인정보 불신** | 약관 루프 4.8% + 로그인 3.5% + 개인정보 0.4% ≈ 9%† | "내 집에 있는 제품들 연결해서 쓰는 건데 굳이 회원가입을 강요하는 이유가 뭡니까" |

† 표시는 **파일럿 5,000건 BERTopic 기준** — 본수집 재분석 후 갱신 예정.

**P-2 측정 방법과 정밀화 이력 (2026-07-21)**
- 초안 정규식은 "제품 등록"의 단순 언급까지 매칭 → 매칭 484건 중 **별점 4~5점이 78건(16.1%)** 혼입
  (예: "제품등록이 쉽네요", "여러 제품을 등록하고 사용 시험 중입니다")
- 샘플 20건 눈검수로 적발 후, **소실·재작업을 명시한 표현만** 남기도록 패턴 축소
  (`재등록`, `첨부터 다시`, `등록…날아/사라/삭제`, `설정…초기화` 등 — [compare_competitor.py](../dx_pipeline_v2.2/compare_competitor.py) 참조)
- 결과: ThinQ 2.62%(448건) → **1.37%(235건)**, 오탐률 16.1% → 11.3%. 경쟁 배수 2.9배 → **2.0배**
- 해석: P-2는 소수 노이즈가 아닌 실재 Pain이나, **P-1에 비하면 부차적**. 헤드라인 1순위는 P-1,
  P-2는 "경쟁 열위가 명확한 차별화 근거"로 배치한다.

**시간축 구조 증거**: 월별 1점 비율 오버레이(`out/compare_rating_trend.png`)에서 2017~2020 구간은
두 앱이 엉켜 있다가 **2021년부터 ThinQ만 지속 상승** — 일시적 장애가 아니라 구조적 열화이며,
"프로필의 클라우드 종속" 진단과 시간축이 일치한다.

한계 명시: ThinQ는 수집 상한 내 **전량**(2015-11~2026-07, 17,446건), SmartThings는 상한 절단
(24,805건, 2017-04~). 비교는 양쪽 공통 구간(2017-04~)으로 정합 후 수행. 2026.07 사태 구간이
포함되어 있으므로 시기 분리 병기 필요.

### 2.2 진단 (분석)
세 Pain의 공통 원인: **홈 프로필(기기 등록·설정·루틴)이 클라우드 계정과 앱에 종속된 구조.** 서버가 죽으면 집이 죽고(P-1), 앱이 리셋되면 설정이 죽고(P-2), 프로필이 서버에 있으니 계정·약관·개인정보 수집이 강제된다(P-3).

### 2.3 처방 (설계 의사결정 — 데이터 아님을 명시)
홈 프로필을 사용자 온바디 디바이스(워치, 캐리어 중립)에 귀속 + 가전과 BLE/UWB 로컬 통신. P-1(서버 무관 동작)·P-2(프로필이 사용자를 따라다님)·P-3(서버가 원본 미보유 = 프라이버시 셀링포인트)을 단일 아키텍처로 해소.
