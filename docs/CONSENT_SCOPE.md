# 동의 범위 — 무계정 로컬 온보딩 (Story 4.1, FR6/AC2)

> **AI 초안 — 최종 책임은 팀.** 이 문서는 온보딩이 요구하는 동의가 **로컬 동작에
> 필요한 범위로 한정됨**을 명시한다(AC2). 코드 원천은 `home_profile/onboard.py`의
> `LOCAL_CONSENT_SCOPE`·`NOT_REQUIRED`이며, `tests/test_onboarding.py`가 이 문서와
> 코드의 정합을 회귀로 고정한다.

P-3 반박(6.2%: *"내 집에 있는 제품들 연결해서 쓰는 건데 굳이 회원가입을 강요하는
이유가 뭡니까"*). 무계정은 **결함이 아니라 셀링포인트**다 — 신원을 넘기지 않아도
집을 연결한다.

## 요구하는 것 (로컬 동작에 필요)

| 동의 항목 | 목적 | 왜 필요한가 |
|---|---|---|
| `ble_pairing` | 가전 로컬 제어를 위한 BLE 근접 페어링 | 워치↔가전 로컬 통신의 물리적 전제 |
| `onbody_storage` | 프로필을 워치(온바디)에 저장 | 프로필 원본이 서버가 아니라 사용자 몸에 귀속(FR7 선행) |

> **정직함은 최소이되 명시다.** "아무 동의도 필요 없다"는 사실이 아니다 — BLE
> 페어링엔 권한이 필요하다. 우리는 필요한 것을 **숨기지 않고 열거**하고, 그 목록이
> 계정·개인정보를 포함하지 않음을 아래 대비로 보인다.

## 요구하지 않는 것 (대비 증거)

| 요구 안 함 | 이유 |
|---|---|
| `account` (계정 생성) | 계정 생성 없이 동작 — 온보딩 경로에 계정 단계가 없다 |
| `login` (로그인·인증) | 로그인 없이 동작 — API에 자격증명 인자가 없다 |
| `email` | 이메일 수집 없음 |
| `phone` | 전화번호 수집 없음 |
| `location` (위치·GPS) | 위치 수집 없음 — 스키마가 `lat`·`lon`·`gps` 키를 거부(FR7) |
| `marketing` | 마케팅 동의 없음 |
| `cloud_backup` | 클라우드 백업 없음 — 원본은 온바디, 서버 미보유(FR7) |

## 구조적 보증

- **네트워크 0**: 온보딩 경로에 서버 호출이 없다. `enforce_offline` 강제 안에서
  온보딩이 성공함으로 증명(`test_onboarding_succeeds_under_offline_enforcement`).
- **식별자 0**: 결과 프로필에 식별자가 0건(`find_identifier_violations` 빈 목록).
  스키마의 `FORBIDDEN_KEY_FRAGMENTS`가 계정·이메일·소유자 키를 이미 거부한다(FR7).
- **동의 감시**: `consent_scope_violations`가 '요구하는 것' 목록에 계정성 항목이
  slip-in 하는 것을 스키마와 같은 어휘로 막는다.

## 심사위원 예상 질문

**Q: "그럼 아무것도 동의 안 받나요?"**
A: 아닙니다. BLE 페어링과 온바디 저장에는 사용자 동의가 필요합니다(위 표). 다만
그 범위가 **로컬 동작에 한정**되며, 계정·이메일·위치·마케팅은 요구하지 않습니다.

**Q: "계정이 없으면 기기 목록은 어디 저장되죠?"**
A: 사용자 워치(온바디)입니다. 서버가 원본을 갖지 않습니다 — 데이터 소재의 상세는
Story 4.2(FR7, 데이터 소재 명시)에서 다룹니다.
