작성 2026-07-22 · Epic 2는 발표의 **마지막 증거**이며(발표 중심 주장 = AI
오케스트레이션, `WORKFLOW.md §5`), 이 문서가 그 실행 대본이다.

## 1. 사전 준비

- [ ] 가민 실기기 충전·페어링 확인 (실기기 데모 시)
- [ ] `python demo_routine.py` 정상 동작 확인 (루프백 폴백)

- [ ] 화면 공유 해상도·폰트 크기 (심사위원이 읽을 수 있게)

## 2. 시연 순서

1. **온바디 저장 보이기** — 프로필이 워치급 크기(1KB대)에 들어감
2. **루틴 실행** — 손목의 프로필이 가전 상태를 바꾼다 (`python demo_routine.py`)
3. **차단 전환** — 아래 §3
4. **차단 상태 재실행** — 같은 코드가 같은 결과를 낸다
5. **대비 자료** — `docs/OFFLINE_COMPARISON.md` (단, ThinQ 칸 미측정 명시)
6. **🌙 클라이맥스 — Night Keeper 야간 모드** (`python demo_night.py --offline`)
7. **재설치 복원 — P-2 반박** (`python demo_reinstall.py --offline`, Epic 3) — 아래 §7
8. **이사 매핑 — P-2 반박(다른 집)** (`python demo_relocate.py`, Epic 3) — 아래 §8
9. **무계정 온보딩 — P-3 반박** (`python demo_onboard.py --offline`, Epic 4) — 아래 §9
10. **데이터 소재 명시 — P-3 반박 마무리** (`python demo_residency.py --offline`, Epic 4) — 아래 §10

### 6. Night Keeper 야간 장면 (발표의 절정)

`python demo_night.py --offline` 한 줄로 Epic 2 전체가 수렴한다:
**"잠들기 전, 네트워크 없이, 손목만으로"** 가전 4대가 밤 상태로 전환.

- Job 문장을 화면에 띄운다: **"내가 잠든 뒤에도 집이 아이를 지킨다"**
- 각 전환이 Job의 어느 부분인지 대응(에어컨 조용히·청소기 끔·조명 끔·냉장고 잠금)
- 페르소나 제시 시 **"행동 프로파일 — 인구통계는 설문 검증 대기"** 병기
  (`docs/PERSONA_NIGHT_KEEPER.md`)
- ⚠️ "집이 아이를 지킨다"는 **서사**다 — child_lock은 시뮬레이터 상태 전이이지
  실제 안전 기능이 아니다(§6 표의 "말하면 안 되는 것"). 안전 보장으로 말하지 않는다

### 7. 재설치 복원 장면 (Epic 3 — P-2 정면 반박, Story 3.1)

`python demo_reinstall.py --offline` 한 줄. 야간 장면(§6)이 "서버 없이 된다"였다면,
이 장면은 **"서버가 없어서 오히려 강하다"**다 — 경쟁 대비 2.0배 열위가 실측된
유일한 지점(P-2 1.37%)을 무효화한다.

- 화면에 실측 리뷰가 뜬다: *"핸드폰 초기화 하고 재설치 하니... 제품이 전부
  없어짐"* (crawl_playstore_thinq.csv 원문 대조 완료 — 이 문장 그대로만 인용)
- 3장면 구성: ①등록 완료(온바디 새김) → ②앱 삭제·재설치(폰은 잊는다,
  워치는 남는다) → ③복원(재등록 0회, 오프라인 강제 안에서)
- 말할 것: "복원 경로에 기기 등록·ref 발급 코드가 **존재하지 않는다** —
  참은 게 아니라 개입할 자리가 없다." 복원의 무(無)클라우드는
  "원본이 서버에 없다"(FR7, Epic 4)의 예고편 — 서사가 여기서 이어진다
- ⚠️ 정직 표기: 참조 어댑터(MemoryCarrier) 기반 — **실기기(가민) 시연 아님**.
  "삼성 껀 그대로인데"의 삼성 비교는 리뷰 원문 인용까지만, 우리 실측 아님

### 8. 이사 매핑 장면 (Epic 3 완성 — P-2 반박의 다른 축, Story 3.2)

`python demo_relocate.py` 한 줄. §7(재설치=같은 집·폰만 초기화)과 **한 쌍**으로
P-2를 완성한다 — 여긴 **다른 집·다른 기기**다. 옛 집에서 길들인 설정을 새 집
기기에 얹되, 안 맞는 건 버리지 않고 **보류**한다.

- 3장면: ①옛 집(에어컨·청소기·조명 + 루틴) → ②이사한 새 집(에어컨 있으나
  fan_speed 미지원·청소기 없음·스타일러 새로 생김) → ③이전/보류 요약
- 말할 것: "매칭되는 건 새 기기로 이전되고, 안 맞는 건 **사유와 함께 보류**된다.
  조용히 버리지 않는 것이 이 기능의 정직성." (이전+보류 = 옛 전체, 누락 0)
- ⚠️ **매칭은 처방(설계)이다** — "device_type 매칭 + capability 교집합"은 우리의
  선택. VOC(P-2)는 문제를 줬을 뿐, 알고리즘의 근거가 아니다
- ⚠️ **H2 정직 표기(AC3):** 페르소나 "전세 거주 Night Keeper"는 **설문 검증
  대기 가설**이다(전세·이사예정 → 온바디 수용도↑, `docs/SURVEY_PLAN.md`).
  발표에서 근거로 쓸 때 "설문 검증 대기" 라벨 병기 필수. 인구통계 날조 금지
  (2.4 김민수·김영희 기각 계보)
- ⚠️ 매핑 시뮬레이션 — 실기기 아님(NFR6)

### 9. 무계정 온보딩 장면 (Epic 4 시작 — P-3 정면 반박, Story 4.1)

`python demo_onboard.py --offline` 한 줄. P-1(서버 없이 동작)·P-2(따라오는 프로필)에
이어 **P-3(개인정보·회원가입 거부, 6.2%)**를 반박한다 — 대표 리뷰: *"굳이 회원가입을
강요하는 이유가 뭡니까"*. 무계정은 결함이 아니라 **신원을 넘기지 않아도 되는 셀링포인트**.

- 화면에 **"계정 0 · 로그인 0 · 클라우드 조회 0"** — 온보딩이 계정 없이 완결됨
- 동의 범위 대비: **요구하는 것**(BLE 페어링·온바디 저장) vs **요구하지 않는 것**
  (계정·이메일·위치·마케팅). 정직함은 최소이되 명시 — 상세는 `docs/CONSENT_SCOPE.md`
- 말할 것: "계정을 참는 게 아니라 만들 자리가 없다 — API에 자격증명 인자가 없고,
  경로에 네트워크 호출이 없다. `--offline`에서 성공하는 게 그 증명." (3.1 AC3 계보)
- ⚠️ FR7 예고: 서버가 원본을 갖지 않음(데이터 소재 명시)은 4.2에서 이어진다
- ⚠️ 참조 어댑터 기반 — 실기기 아님(NFR6)

### 10. 데이터 소재 장면 (Epic 4 마무리 — P-3 반박 완성, Story 4.2)

`python demo_residency.py --offline` 한 줄. §9(무계정)와 프라이버시 한 쌍 —
"신원을 안 넘긴다"(4.1) + **"그럼 데이터는 어디"**(4.2)에 답한다. "서버에 없다"를
말이 아니라 **화면으로 증명**한다.

- 화면에: **원본 위치=온바디 · 서버 원본 보유=아니오 · 서버 전송 항목=없음 ·
  온바디만으로 복원=예**. 마지막 항목이 "원본이 온바디에 있다"의 증거(3.1 재사용)
- 온바디 footprint(레코드 종류·개수·바이트)를 보인다 — 단 device_ref 원문은
  노출하지 않는다(이름은 PII 운반체일 수 있다)
- 말할 것: "온바디만으로 프로필이 복원됩니다 — 다른 저장소가 필요 없다는 관찰이고,
  소재 확인 자체가 오프라인 강제 안에서 성공합니다." 상세는 `docs/DATA_RESIDENCY.md`
- ⚠️ **한계 정직 표기**: 이 증명은 우리 코드 범위(참조 어댑터·이 프로세스)까지.
  벤더 SDK·OS 텔레메트리는 범위 밖 — "서버에 없다"를 무조건형으로 과장하지 않는다
- ⚠️ 참조 어댑터 기반 — 실기기 아님(NFR6)

## 3. 기내모드 전환 시점

> [!important] 두 층위의 차단을 구분해서 보인다
> - **프로세스 차단** (`--offline`): 코드가 증명한다. 화면에 "소켓 열기 시도 → 실패 확인"
> - **장비 차단** (기내모드): **사람이 폰/노트북에서 누른다.** 코드가 증명하지 못한다

시연 순서:
1. `--offline` 실행 → 화면에서 하네스 차단 시연을 보인다
2. **그 다음** 발표자가 장비 기내모드를 켠다 (관객이 보는 앞에서)
3. 기내모드 상태에서 루틴을 다시 실행 → 성공

## 5. 예상 질문과 답

**Q: "서버가 없다면서 PostgreSQL은 뭐죠?"**
A: 저장소엔 트랙이 둘입니다. **발견** 트랙(공개 리뷰 VOC 분석)과 **처방**
트랙(온바디 프로필). DB는 발견 트랙에만 있고, 처방 트랙은 DB를 보지 않습니다.
그 경계는 AST 테스트로 코드에서 강제됩니다.
(근거: `docs/meetings/2026-07-22_postgres-portfolio-track.md`)

**Q: "정말 서버 통신이 없는 걸 어떻게 증명하죠?"**
A: 두 각도입니다 — 코드에 네트워크 경로가 없음을 정적(AST)·동적(호출 차단)으로
고정했습니다. 다만 이것은 "우리 코드가 못 나간다"까지이고, 패킷 수준 증명은
아닙니다. 아래 §6의 구분을 지킵니다.

**Q: "가민 아닌 워치에서도 되나요?"**
A: 구조는 캐리어 중립입니다(1.3). 어댑터 추가만으로 확장되며, 실증은 가민
하나로 했습니다. 미구현 캐리어는 "미구현"으로 정직 표기됩니다.

## 6. ⚠️ 발표 중 말해도 되는 것 / 안 되는 것

`docs/OFFLINE_EVIDENCE.md §2`에서 가져옴 — **즉흥 과장을 막는 장치**:

| 말해도 되는 것 | 말하면 안 되는 것 |
|---|---|
| "우리 코드에 네트워크 경로가 없음을 테스트로 고정했다" | ❌ "네트워크 캡처로 확인했다" |
| "차단 상태에서 동일한 상태 전이가 나옴을 보였다" | ❌ "서버 통신이 없음을 증명했다"(무조건형) |
| "ThinQ 앱 대비는 아직 미측정" | ❌ "ThinQ는 안 됩니다"(측정 없이 단정) |

## 7. 실패 시 폴백

- **BLE 안 뜨면** → 루프백으로 시연. 화면에 "루프백 — BLE 아님" 정직 표기 유지.
  구조 설명은 그대로 성립한다 (발표 범위는 이미 "가민 → BLE → 시뮬레이터")
- **오프라인 하네스 문제** → `--offline` 없이 실행 + 장비 기내모드로 대체
- 어느 경우에도 **동작한 척 금지** — 폴백을 썼으면 폴백을 썼다고 말한다 (NFR6)
