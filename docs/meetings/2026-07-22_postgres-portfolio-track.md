## 3. 붙일 자리 — 후보 3개 중 B+C

| | 내용 | 판정 |
|---|---|---|
| A | `01_collect` 뒤 원본 적재소 | ❌ CSV를 테이블에 부은 것뿐. "왜 DB냐"에 답 없음 |
| B | 분석 결과(세그먼트·페인포인트) 질의 | ✅ 채택 |
| C | 실행 이력(`run_id`·해시·게이트·에이전트 결과) | ✅ 채택 |

**B+C는 원래 한 몸이다** (Amelia):
```sql
WHERE run_id = (SELECT max(run_id) FROM ops.runs WHERE gate_status='pass')
```
C가 없으면 B의 이 절이 성립하지 않고, B가 없으면 C는 조회할 사람이 없다.
*"게이트 통과한 실행의 데이터만 분석한다"* — 계보 서사가 SQL에 그대로 실린다.

C의 근거 보강 (Winston): C는 **DB가 아니면 못 하는 일**이다 — 실행 30번의 시계열
질의("게이트 실패율이 언제부터 올랐나"), 실행 간 비교("run A와 B에서 P-1 언급률
차이"). 폴더 타임스탬프로는 불가능하다. 그리고 이건 **우리의 실제 아픔**이다 —
2026-07-22 하루에만 bmad config가 11번 뒤집혔고 실행 이력이 어디에도 안 남는다.

## 4. "왜 DB인가"의 정정 — 면접 답변까지 포함

사용자 주장: *"현업에선 예산 문제로 DB 많이 쓰지 않나? 안 그러면 용량이 너무 크잖아"*

**절반 맞고 절반 정정** (Mary·Winston):

- ✅ **맞는 절반**: 현업 데이터는 DB/웨어하우스에 산다. 분석가 실무 첫날 받는 건
  CSV가 아니라 웨어하우스 접속 계정이다. 포폴이 실무처럼 보이는 것 자체가 효용이다
  → Mary는 앞서 "B는 한계효용 낮음"이라 했던 판정을 **철회**했다
- ❌ **정정할 절반**: 현업이 DB를 쓰는 이유는 **용량 절약이 아니다.** 저장 단가만
  보면 파일이 더 싸고, 그래서 진짜 대용량은 RDB가 아니라 데이터 레이크(S3+Parquet)로 갔다

**진짜 이유 4가지** — 면접에서 "왜 DB 쓰셨어요?"에 대한 답:

| 이유 | 뜻 |
|---|---|
| **동시성** | 분석가 여럿이 같은 테이블을 동시 질의 |
| **단일 진실 원천** | 팀마다 다른 CSV 사본으로 답하는 참사 방지 |
| **권한·거버넌스** | 행·열 단위 접근 통제 |
| **선언적 질의** | "어떻게 읽을지"가 아니라 "뭘 원하는지" |

> ⚠️ **"용량 때문에요"라고 답하면 감점.** "동시 접근과 단일 진실 원천 때문에요"가 정답.

**우리 저장소가 그 참사의 축소판이다** (Mary): `data/`에 `crawl_naver.csv`,
`crawl_naver_v1.csv`, `crawl_naver_yield.csv`, `naver_valid.csv` — 같은 소스의 사본이
넷이고 "어느 게 진짜냐"가 파일명 추리 게임이 돼 있다.
→ B의 서사는 "SQL 해봤다"가 아니라 **"사본 지옥을 단일 진실 원천으로 정리했다"**다.
그러면 15행짜리 `painpoints` 테이블이어도 시시하지 않다.

## 6. 호스팅 — Docker

| | 장점 | 단점 |
|---|---|---|
| 로컬 설치 | 의존성 최소, 성능 | 머신에 눌러앉음, 재현 불가, 저장소에 남길 게 없음, Windows 인코딩 지옥 |
| **Docker** ✅ | `compose.yml` 한 장으로 재현, `down -v`로 흔적 없이 원복, 버전 고정, UTF-8 기본, **환경도 파일로 영구화**(pytest 자산화 원칙과 동일 계보) | Desktop 무거움, WSL2 필요 |
| 클라우드 무료 티어 | 어디서든 접속, 브라우저 SQL 시연 | **공개 저장소 + 접속 문자열 = 유출 사고 1순위**, 무료 프로젝트 소멸(포폴은 몇 달 뒤 열리는 물건), Sally의 "서버 없다면서" 오해를 자초 |

**결정: Docker 기본.** 되돌리기 쉬운 것을 먼저 고른다(Winston) — 접속 문자열만
갈아끼우면 나중에 옮길 수 있는 결정이므로.
**포폴 공개 시점에 클라우드 읽기 전용 사본을 추가**하는 옵션은 열어둔다(Mary) —
스키마·적재 스크립트가 같으니 둘 다 돌아간다.

### ⚠️ Docker 실측 (2026-07-22, 정직 표기)

| 항목 | 상태 |
|---|---|
| Docker CLI | ✅ v29.5.3 설치됨 |

Windows Docker는 WSL2 백엔드가 필요하고 그것이 이 머신에서 되는지는 **미확인**이다.
Epic 5 착수 시점에 **연결 스파이크 먼저**(Paige) — 결정보다 실측이 앞선다.
